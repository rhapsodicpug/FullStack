# How to Find Your Deployment URLs

## Backend URL (Vercel)

### Method 1: From Vercel Dashboard

1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Click on your **backend project**
3. Look at the top of the page - you'll see:
   - **Production URL**: `https://your-project-name.vercel.app`
   - This is your backend URL!

### Method 2: From Deployments Tab

1. Go to **Deployments** tab
2. Click on the latest deployment (the one with a green checkmark)
3. You'll see the URL at the top, something like:
   - `https://your-project-name-abc123.vercel.app`
   - Or `https://your-project-name.vercel.app`

### Method 3: From Project Settings

1. Go to **Settings** â†’ **General**
2. Look for **Domains** section
3. Your production domain will be listed there

## Example URLs

Your backend URL will look like:
```
https://fullstack-auth-backend.vercel.app
```
or
```
https://fullstack-auth-backend-abc123.vercel.app
```

## Using the Backend URL

Once you have your backend URL, use it in:

### Frontend `.env.local`:
```env
NEXT_PUBLIC_API_URL=https://your-backend-url.vercel.app/api
```

### Testing Endpoints:
- Root: `https://your-backend-url.vercel.app/`
- Health: `https://your-backend-url.vercel.app/api/health`
- Signup: `https://your-backend-url.vercel.app/api/auth/signup`
- Login: `https://your-backend-url.vercel.app/api/auth/login`

## Note

- The URL is automatically generated by Vercel
- It's based on your project name
- You can add a custom domain later if needed
- The URL is publicly accessible (no authentication needed to view it)
